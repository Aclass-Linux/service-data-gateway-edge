## 一、项目总体目标

开发一个**工业智能网关原型**，能够：

- **采集数据**：模拟或真实读取现场设备数据（温度、振动、PLC寄存器）
- **边缘处理**：数据清洗、阈值报警、本地规则引擎
- **远程通信**：通过MQTT上传至云平台（如阿里云IoT、EMQX）
- **远程配置**：支持通过网络修改采集参数和报警阈值
- **本地监控**：提供Web状态页面或命令行管理界面

**最终产出**：一个完整的、可部署的网关软件包，能写在简历上作为核心技术项目。

## 四、分阶段详细计划

### 第一阶段：单进程采集 + 上传原型（第1-6周）

**目标**：跑通"采集 → 上传"最小链路，掌握Linux基础IPC和网络。

| 周次 | 学习主题 | 网关具体任务 | 产出物 |
| :--- | :--- | :--- | :--- |
| 第1周 | 文件IO、信号、定时器 | 实现模拟数据源：每秒生成随机温度值，写入命名管道（FIFO） | `simulator` 程序 |
| 第2-3周 | 进程、管道、信号处理 | 主控程序fork采集子进程，通过管道接收数据并打印 | `gateway_v1` 基础框架 |
| 第4周 | Socket TCP客户端 | 增加MQTT发布模块，将数据上传至公共MQTT broker | 数据同时打印+上传 |
| 第5周 | 守护进程、syslog | 主控程序daemon化，日志记录到syslog，支持优雅退出 | 可作为系统服务运行 |
| 第6周 | 调试与稳定性 | 增加看门狗（子进程异常退出时自动重启），学习core dump分析 | 稳定版单进程网关 |

**里程碑**：完成一个完整的工业物联网南向+北向链路（数据模拟）。

---

### 第二阶段：多线程边缘处理（第7-11周）

**目标**：引入现代C++，实现多线程、数据缓冲和规则引擎。

| 周次 | 学习主题（C++特性） | 网关改进任务 | 产出物 |
| :--- | :--- | :--- | :--- |
| 第7周 | 类、RAII、智能指针 | 用C++重写主控：`DataPoint`类、`RingBuffer`类 | 内存安全的基础框架 |
| 第8-9周 | 标准线程库、互斥锁、条件变量 | 实现生产者-消费者模型（采集→缓冲→处理线程池） | 高吞吐解耦数据流 |
| 第10周 | 简单模板、lambda | 增加规则引擎：支持配置文件定义报警规则 | 可配置的边缘计算 |
| 第11周 | 移动语义、现代网络库 | 重构MQTT模块，支持异步重连，使用`std::move`传递数据 | 健壮的上传模块 |

**里程碑**：网关具备边缘计算雏形，可处理每秒上千条数据点。

---

### 第三阶段：远程管理能力（第12-14周）

**目标**：让网关可被远程配置和监控，具备产品化基础。

| 周次 | 学习主题 | 网关任务 | 产出物 |
| :--- | :--- | :--- | :--- |
| 第12周 | HTTP协议、libmicrohttpd库 | 实现配置服务器：提供REST API（GET/POST /config） | 可通过浏览器管理的网关 |
| 第13周 | 配置文件解析（libconfig/JSON） | 配置持久化到文件，支持SIGHUP热加载 | 配置动态生效 |
| 第14周 | SQLite基础 | 增加本地数据缓存：MQTT断连时存入数据库，恢复后补传 | 断网续传能力 |

**里程碑**：网关具备商业化产品的基础功能。

## 五、Rust学习安排（提前到第4-6个月）

### 为什么不用等两年？

1. **工业智能网关需要内存安全 + 高并发**：Rust的所有权系统正好解决数据竞争问题
2. **你已有C语言基础**：理解内存布局，学Rust比纯新手快
3. **ROS 2已支持Rust**（`rclrs`库）：未来机器人领域机会更多
4. **全职学习的时间窗口有限**：趁学习热情高，一口作气掌握C++和Rust

### Rust学习路径（第4-6个月，穿插在网关项目间隙）

| 时间 | 学习内容 | 具体做法 | 网关项目结合 |
| :--- | :--- | :--- | :--- |
| **第4个月** | 《The Rust Book》前10章 | 所有权、结构体、枚举、错误处理、泛型、trait | 无（纯语法练习） |
| **第5个月** | FFI（外部函数接口） | 学习Rust编译为动态库供C++调用 | 用Rust写串口采集模块，C++主控调用 |
| **第6个月** | 所有权+生命周期对比 | 对比C++移动语义与Rust所有权 | 用Rust重写环形缓冲区，对比C++版本性能 |

### 第4-6个月的具体任务表

| 周次 | Rust学习任务 | 验收标准 |
| :--- | :--- | :--- |
| 第17周 | 安装Rust工具链，完成《Rust Book》第1-6章 | 能写简单的猜数字游戏 |
| 第18周 | 完成《Rust Book》第7-10章（模块、集合、错误处理、泛型） | 能实现一个泛型`Result<T,E>`处理函数 |
| 第19周 | 学习trait和闭包 | 为`DataPoint`结构体实现`Display` trait |
| 第20周 | 学习FFI（`extern "C"`、`#[no_mangle]`） | 写一个Rust函数`add(a,b)`，从C语言调用 |
| 第21周 | 实现Rust采集模块（FFI） | Rust读取模拟温度传感器，C++主控通过FFI获取数据 |
| 第22周 | 用Rust重写环形缓冲区 | 对比C++版本的线程安全性和内存开销 |
| 第23-24周 | 缓冲：复习+整合 | 将Rust模块集成到网关主程序 |

## 六、编译原理学习建议（放到18个月后）

### 为什么要等？

1. **对网关项目无直接帮助**：简单规则引擎用表达式解析库（如`muParser`、`exprtk`）即可
2. **学习曲线陡峭，收益滞后**：需要大量理论和算法基础
3. **与转型目标关联弱**：机器人/物联网岗位很少要求编译原理

### 两阶段学习法

#### 第一阶段：实用为主（第6-12个月，穿插学习）

**目标**：掌握与日常开发相关的编译/链接知识，不深入算法。

| 学习内容 | 具体做法 | 预计时间 |
| :--- | :--- | :--- |
| 链接、装载、ELF文件格式 | 读《程序员的自我修养》前6章 | 2周（每天30分钟） |
| 符号表、重定位、动态链接 | 用`readelf`、`objdump`分析C++/Rust编译产物 | 1周 |
| 递归下降解析 | 为网关规则引擎手写算式求值器（支持`a>10 && b<5`） | 2周 |

#### 第二阶段：系统学习（18个月后，按需深入）

**只有在以下情况下才需要**：
- 职业方向确定为：数据库内核、编程语言设计、静态分析工具
- 你对编译器有持续热情，愿意投入200+小时

**学习路径**：
1. 通读《编译原理》（龙书）第1-6章
2. 实现玩具编译器（如`c--`到汇编）
3. 学习LLVM框架基础

## 八、软硬件准备

### 最低配置（零成本，立即开始）

- Ubuntu 22.04虚拟机（4GB内存，支持嵌套虚拟化）
- 用 `socat` 模拟串口设备：`socat -d -d pty,raw,echo=0 pty,raw,echo=0`
- 用 `mosquitto` 本地搭建MQTT broker，或使用公共测试broker（`broker.emqx.io`）
- QEMU用户态模拟（用于ARM交叉编译测试）

### 推荐配置（300-500元，体验更佳）

- 树莓派4B 2GB 或 国产RK3566开发板
- USB转485/232模块
- 阿里云IoT平台免费试用实例
- 二手PLC（如西门子S7-200）用于真实Modbus测试

### Rust开发环境

```bash
# 安装Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 安装交叉编译目标（用于ARM）
rustup target add armv7-unknown-linux-gnueabihf

# 安装工具
cargo install cargo-edit cargo-expand
```

## 十、常见问题与提醒

### 1. 学习策略

- **不要跳过项目**：即使某周理论没看完，也要把代码跑通
- **从简单开始**：第一版可以不支持多线程、不处理所有异常，后续迭代改进
- **保持迭代**：每周末提交一次代码，写下下周改进计划

### 2. 调试工具

- `gdb`：调试段错误、死锁
- `valgrind`：检测内存泄漏
- `strace`：追踪系统调用
- `perf`：性能分析
- `cargo check`：Rust快速语法检查

### 3. Rust学习常见陷阱

- **不要试图用Rust写所有模块**：先用FFI与C++主控集成，逐步替换
- **所有权系统需要时间适应**：前2周会很痛苦，坚持写小例子
- **善用编译器提示**：Rust的错误信息非常友好，仔细阅读

### 4. 与工作结合

即使工作中写传统协议，也尝试用本项目中的技术去重构：
- 用C++类和RAII管理资源
- 用多线程分离收/发/处理逻辑
- 用Rust写安全的解析器

**这些行为会自然成为面试中的亮点**。

### 5. 时间管理

- 如果某阶段任务过重，优先保证：网关主线（C++/Linux）> Rust > ROS/Yocto
- 编译原理相关内容可以用碎片时间学习（通勤、午休）

## 附录：快速命令参考

### 常用调试命令

```bash
# 查看进程
ps aux | grep gateway

# 查看日志
tail -f /var/log/syslog | grep gateway

# 测试MQTT
mosquitto_pub -t "test" -m "hello"
mosquitto_sub -t "#" -v

# 查看ELF信息
readelf -h gateway
objdump -d gateway | less

# Rust相关
cargo build --release
cargo test
cargo expand  # 查看宏展开
```
