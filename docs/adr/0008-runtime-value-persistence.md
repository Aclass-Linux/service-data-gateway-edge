# ADR 0008: 运行时值持久化（脏页位图 + 页粒度落盘 + seqlock 读）

参见 ADR-0009（总线值表示）：持久化槽位 `_Atomic uint64_t` 与 `egw_value_t.raw` 共用同一位模式，seqlock 直接操作 `raw`。

## 状态

已接受（与 ADR 0003 点表、ADR 0007 线程模型协同）

## 背景

点表拓扑（采集参数、寄存器地址、sig_id）是离线构建、运行期只读的（ADR 0003）。但网关运行中还有一类**可变值**需要持久化：测点最新采集值、累计量/计数器、标定后的 scale/offset 等。这些值需要：

1. 主回路高频更新，**绝不能引入阻塞**（不能在事件循环线程做磁盘 I/O）
2. 跨重启恢复：开机能读回上次的值作为初始状态
3. 可容忍丢失最后若干秒（断电时尚未落盘的更新），属于「尽力持久化」语义，不是强一致交易数据
4. 可见性：能看出最近一次落盘发生在何时

强一致、不可丢的数据（时序补传、审计日志）仍走 SQLite 时序缓存，不在本 ADR 范围。

## 决策

### 1. 内存当前值表为真相，文件为持久化后端

主回路在内存中维护当前值表 `egw_live_slot_t live_table[]`，按 `(device_id, sig_id)` 全局索引、定长槽位密集排列。槽位分配在离线工具构建时完成，路由表中存储每个 `(device_id, sig_id)` 对应的槽位偏移。内存是真相，运行时文件是它的周期性快照。运行期高频读写都走内存，不碰磁盘。

文件结构：
```
[file header: magic, version, build_id, last_flush_unix_ms]
[global slot array: all (device_id, sig_id) pairs, indexed by routing table]
```

### 2. 单独的持久化线程独占写

落盘逻辑放在专属持久化线程，独占运行时文件的写权限和 fd，所有阻塞 I/O（`pwrite`/`fdatasync`）都在此线程。主回路只通过置脏位（见下）告知「哪些数据变了」，零阻塞。线程间唤醒沿用 ADR 0007 的 uv 投递 / 定时器模型。

### 3. 脏页位图跟踪

主回路更新某个 sig 值时，顺带把它所在页的 dirty bit 置 1：

```
dirty_bitmap[(slot_offset / PAGE_SIZE) / 8] |= 1u << ((slot_offset / PAGE_SIZE) % 8);
```

置位是一条按位或，几乎零成本。每 4KB 数据只占 1 bit。

### 4. 页粒度落盘（PAGE_SIZE = 4KB）

持久化线程定期扫描脏页位图，**只对置位的页做整页 `pwrite`**，落盘单位是 4KB 页，而非单值或全量：

- 对齐文件系统块和 flash 擦写单元，控制写放大（边缘网关多为 eMMC/SD/Nand，有擦写寿命问题）
- 系统调用次数 = 脏页数，远小于脏值数；同页内多个点变脏合并为一次写
- 一页只要有一个值变就整页落，是页粒度的预期行为

落盘顺序保证时间戳不领先于数据：先 `pwrite` 脏页数据 → `fdatasync` → 再更新文件头 `last_flush_unix_ms` → 再 `fdatasync`。崩溃在中间最坏是时间戳偏旧，数据不会半截。`last_flush_unix_ms` 同时供开机判断数据新鲜度，并在落盘后打一条日志（写了多少页、何时），满足可见性需求。

### 5. seqlock 读 + 有界短延迟重试

落盘读取单个 sig 槽位时用 seqlock 检测撕裂。槽位含一个偶/奇代数 `gen`，主回路写值时 `gen` 先变奇、写完变偶：

```c
for (int retry = 0; retry < 3; retry++) {
    uint32_t g1 = load_acquire(&s->gen);
    if (g1 & 1u) { short_sleep(); continue; }   /* 主回路正在写 */
    uint64_t v = load_acquire(&s->value);
    uint32_t g2 = load_acquire(&s->gen);
    if (g1 == g2) { /* 一致快照，写入页缓冲 */ break; }
    short_sleep();                               /* 读期间被改，重试 */
}
```

重试作用在**单个 sig 槽位**层面（不是整页——一页几百个点，按整页重试会因总有点在变而永远落不下去）。

「比较短的时间」用 `nanosleep` 量级（约 50µs）让出 CPU，而非纯自旋——持久化线程与主回路在不同线程，单核 ARMv7 上纯自旋会与主回路抢 CPU。最多 3 次约 150µs，可忽略。

**3 次仍读不到一致值的 sig**：用该槽位上一次已知的稳定值落盘，并**保留它的脏位不清**，下个周期必然再次尝试。落盘是快照语义，延迟一个周期可接受，最终值不丢。其余 sig 与该页正常落盘。

成功落盘的页清除其脏位。

## 否决的方案

**单值粒度落盘（每个脏 sig 一次 pwrite）**：写放大最小，但 N 个脏点 = N 次系统调用 + 离散小写，对 eMMC/机械盘不友好。

**全量落盘（每周期整个变量区重写）**：实现最简，但大部分未变数据被重写，写放大和 flash 磨损严重。

**MAP_SHARED + msync 由主回路触发**：`msync(MS_SYNC)` 阻塞，违反主回路零阻塞；`MS_ASYNC` 落盘时机不可控。落盘必须隔离到独立线程。

**直接复用 SQLite 作运行时值后端**：功能足够，但运行时值是定长、高频、可容忍丢失的快照数据，自实现脏页位图 + 页落盘更轻、零额外查询开销；SQLite 留给强一致的时序缓存与审计日志。

**纯自旋等待（cpu_relax 无限重试到一致）**：单核 ARMv7 上会与主回路抢 CPU，高频改写的 sig 可能让持久化线程长时间空转。改为有界 3 次 + 短睡眠 + 下周期重试。

**整页 seqlock 重试**：一页几百个点，只要有一个在写就重试整页，会导致热点页永远落不下去。重试下沉到单槽位。

## 理由

- **主回路零阻塞**：更新值 = 内存写 + 一条按位或置脏位；无锁、无系统调用。
- **脏跟踪粒度 = 落盘粒度**：位图天然按页，扫描即得脏页，无需额外归并。
- **写放大可控**：页粒度对齐 flash 擦写单元，保护边缘设备存储寿命。
- **读一致无锁**：seqlock 让持久化线程读到完整快照，主回路写路径不被读路径拖慢。
- **最终值不丢**：3 次失败保留脏位，下周期补落，配合「尽力持久化」语义。

## 后果

- 需定义 `egw_live_slot_t`（`_Atomic` 值 + `gen` 代数）与脏页位图，sig_id → 槽位偏移复用全局路由表索引（ADR 0005）。
- 运行时可变值约束在 64 位以内（`_Atomic uint64_t` + union 解释类型）；更宽数据不进此路径。
- 持久化线程需实现页扫描、seqlock 读、`pwrite`/`fdatasync` 落盘与 `last_flush_unix_ms` 文件头更新。
- 开机时持久化线程读回运行时文件，填充内存当前值表作为初始状态。
