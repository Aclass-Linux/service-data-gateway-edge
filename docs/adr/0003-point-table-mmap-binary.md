# ADR 0003: 点表使用 mmap 二进制文件

## 状态

已接受

## 背景

设备测点清单（点表）描述如何在线缆上解析/编码字节，以及本侧的值处理系数（南向：功能码、寄存器地址、南向交互类型、采集周期、本侧 scale/offset 与死区；北向：北向目标类型、通道、地址映射、本侧 scale/offset 与死区）。点表需要：

1. 零拷贝访问：运行时高频读取，不能每次查表都做 JSON 解析或磁盘 I/O
2. 与路由表同源构建：点表、北向点表、`routing.bin` 来自同一次离线构建，路由表是 sig_id 的对齐契约
3. 多设备隔离：每设备独立南向点表文件，便于后续按设备做更新

转换系数与死区遵循「谁用谁存」，由南北向点表各自持有；路由表只存索引、双向地址映射和核心数据类型（ADR-0005）。

## 决策

采用 `.bin` 二进制格式存储点表，运行时通过 `mmap` 只读访问。

每个 `.bin` 文件头包含：
```c
struct egw_bin_header {
    uint32_t magic;        /* 文件类型标识（南向/北向/路由表各不同） */
    uint32_t version;      /* 格式版本号 */
    uint32_t build_id;     /* 离线工具构建时生成的唯一 ID（时间戳或 UUID） */
    uint32_t checksum;     /* 文件内容校验和（CRC32 或类似） */
    uint8_t  endianness;   /* 字节序标记（0x01=小端, 0x02=大端） */
    /* ... 其他元数据字段 ... */
};
```

网关启动时验证：
1. **同源构建**：南向点表、北向点表、路由表三个文件的 `build_id` 必须相同（保证地址映射一致）
2. **完整性**：校验和匹配（文件未损坏）
3. **平台兼容**：字节序匹配当前平台
4. **格式兼容**：`version` 与代码期望的格式版本兼容

任一验证失败则拒绝启动，防止运行时出现地址错乱或数据损坏。

第一版采用启动时加载、运行期只��的自定义紧凑 C 结构体数组 + `mmap` 映射二进制文件（`.bin`）。每设备一个独立南向点表文件；北向点表与全局 `routing.bin` 一起由离线工具生成。

本 ADR 的「只读」指**点表拓扑**（reg_addr、功能码、数据类型、采集周期、sig_id）只读。运行中会变化的可变值（最新采集值、计数器、标定参数）不存放于此，而由独立的运行时值持久化路径管理，见 ADR 0008。

第一版不支持运行期增删 `sig_id` 或单独替换某个点表文件。点表和 `routing.bin` 必须来自同一次离线构建，作为一致快照部署。

后续版本可以在此基础上增加热更新：通过版本号/校验和确认南向点表、北向点表和 `routing.bin` 属于同一快照，再整体原子切换 mmap 映射。热更新不是第一版能力。

结构体使用固定宽度类型（`uint32_t`、`int16_t`）和 `__attribute__((packed))`，确保 x86_64 和 ARMv7 对齐一致。

字节序固定为**小端**。文件头包含 magic、版本号和字节序标记，加载时校验：端序或版本不符直接拒绝加载（快速失败），而非静默读出错误地址。目标平台（x86_64、主流 ARMv7）均为小端，固定小端可零转换直接 mmap 当数组用；代价是放弃大端平台的零拷贝（大端机需加载时整体转换），当前目标平台无此需求。

## 否决的方案

**JSON 每帧解析**：运行时查询 JSON 树需要字符串解析和 cJSON API 调用，高频采集场景下 CPU 开销不可接受。

**SQLite 存储点表**：支持复杂查询和更新，但每次读取需要 B-tree 遍历和系统调用，不如 mmap 的直接内存访问快。且 SQLite 主要已用于时序缓存，职责分开。

**FlatBuffers**：支持零拷贝和 schema 演进，但需要引入 FlatBuffers 编译器作为构建依赖，且运行时库体积较大。当前场景字段固定，schema 演进需求不强。

## 理由

- **零开销访问**：mmap 后文件内容即内存，`points[dev_id][idx].reg_addr` 等价于数组下标访问，无系统调用、无序列化开销
- **OS 管理页缓存**：文件由内核 page cache 管理，不主动访问不占物理内存
- **一致快照**：第一版点表和 `routing.bin` 同源生成、启动加载、运行期只读，避免单文件热替换导致 sig_id 孤儿或死路由
- **跨平台对齐一致**：固定宽度类型 + packed 属性保证结构体布局一致；文件字节序需由格式头显式声明
